# 동영상 자동 모자이크 구현 로드맵

GPU 가속 기반 동영상 얼굴·번호판 자동 모자이크 기능의 단계별 구현 계획.
`CODE_REVIEW_2026-07-02.md`의 장기 로드맵 항목("GPU 가속 → 동영상 지원")을 구체화한 문서.

## 목표

- 동영상을 드롭하면 얼굴/번호판이 **모든 프레임에서 빠짐없이** 모자이크된 사본을 출력한다.
- 모자이크가 깜빡이거나 떨리지 않는다(시간적 일관성).
- 오디오·화질·메타데이터 정책은 사진과 동일한 원칙: 원본 불변, 기본 메타데이터 제거, 로컬 처리.

## 핵심 설계 결정

| 결정 | 근거 |
|---|---|
| 탐지 모델은 기존 SCRFD·YOLOv9-t 유지 | 추적은 모델이 아니라 알고리즘으로 해결. 새 모델 다운로드가 없어 기존 SHA-256 공급망 검증 체계를 그대로 유지 |
| 추적은 ByteTrack식 detection+association 자체 구현 | 칼만 필터 + IoU 매칭 순수 알고리즘(신경망·외부 의존성 없음). 저신뢰 탐지를 트랙 연결에 활용하는 방식이 모션 블러·옆얼굴 프레임 복구에 직결 |
| end-to-end MOT(MOTR/MOTIP/SAM2 계열)는 채택하지 않음 | 우위가 ID 일관성에 있는데 모자이크는 ID가 아니라 리콜이 중요. 얼굴용 사전학습 모델 부재, ONNX 내보내기 곤란, 수백 MB급 무게 |
| 2-pass 오프라인 처리 (분석 → 적용) | 정·역방향 트랙 합집합, 공백 보간, 좌표 스무딩으로 "한 프레임 유출"을 구조적으로 차단. 실시간 처리는 비목표 |
| 비디오 I/O는 FFmpeg CLI 서브프로세스 파이프 | OpenCV VideoWriter는 오디오가 사라져 탈락. libav 직접 링크 대비 빌드·패키징 단순, GPL-3.0 앱이라 라이선스 문제 없음 |
| v1은 동영상 리뷰 UI 없음 | 프레임 단위 리뷰는 동영상에 부적합. 트랙 단위 타임라인 리뷰는 후속 릴리스로 분리 |

## Phase 0 — 기반 정리 (선행 작업)

코드리뷰 중기 로드맵과 동일. 동영상 파이프라인이 얹힐 토대.

- [x] `redactly_core` 정적 라이브러리 추출 — 탐지기·Mosaic·Tracking·ImageIo·ProcessorWorker를
  UI(Widgets)에서 분리(core는 Qt Core/Gui까지만 의존). 테스트 타깃들이 소스 재컴파일 대신 core를 링크.
  하드닝 플래그를 `redactly_harden()` 함수로 추출해 core에도 적용(기존에 앱 타깃 한정이던 코드리뷰 지적 해소)
- [ ] `MainWindow.cpp`에서 ModelCatalog/ModelDownloader 추출 — 동영상 릴리스에서 이 파일이 더 커지는 것을 방지
- [ ] 이미지 병렬 처리 도입 — 여기서 만드는 프레임 단위 작업 분배 구조를 동영상 파이프라인이 재사용

**완료 기준**: 기존 이미지 테스트 전체 통과, 앱 동작 회귀 없음.

**결정 사항 (2026-07-03)**: FFmpeg는 번들로 확정 — 시스템 의존 대비 인코더 구성·버전이
보장되어 모든 사용자에게 검증된 동일 결과를 제공. 미해결 질문 1번 해소.

## Phase 1 — GPU 가속 (코드리뷰 추천 기능 #5, 규모 M) — ✅ 구현 완료 (dev 브랜치)

동영상의 전제 조건이지만 사진 사용자에게도 즉시 이득이므로 독립 릴리스 가능.

- [x] `Ort::SessionOptions`에 실행 프로바이더 추가 — `OrtAcceleration.cpp`의 `applyOrtAcceleration()`으로 공통화
  - [x] macOS: CoreML EP (MLProgram) + `com.apple.security.cs.allow-jit` entitlement
  - [x] Windows: DirectML EP 런타임 감지(있으면 사용) — 단, 현재 CI가 CPU 전용 ONNX Runtime 1.20.1을 받으므로 DirectML 동봉 패키지로의 전환은 후속 작업
  - Linux: v1은 CPU 유지 (CUDA/ROCm은 배포 부담 대비 수요 확인 후)
- [x] EP 초기화 실패·부재 시 CPU 자동 폴백 + 로그 기록, 실행 로그에 백엔드 표시
- [x] 설정에 가속 on/off 토글 (기본 on) — 토글 변경 시 캐시된 탐지기 세션 무효화
- [x] 벤치마크 (Apple Silicon, 640px 입력, 10회 평균): SCRFD 2.5g 83.3ms → 6.6ms(12.6×), SCRFD 10g 279.8ms → 31.3ms(8.9×)

**완료 기준 충족**: CPU 폴백 동작 확인, 위 벤치마크 기록. 미완: 공증(notarization) 통과 확인은 서명 가능한 릴리스 환경에서 필요.

## Phase 2 — 트래킹 엔진 (비디오 I/O와 독립, 병행 가능) — ✅ 구현 완료 (dev 브랜치)

순수 로직이라 동영상 파일 없이도 이미지 시퀀스로 개발·테스트 가능. 가장 먼저 테스트가 갖춰져야 하는 부분.

- [x] `redactly::Track`/`TrackedBox` 타입 (`Tracking.hpp`): 트랙 ID, 프레임별 박스, 보간 여부
- [x] ByteTrack식 2단계 연결기 (`ByteTracker`)
  - 고신뢰 탐지 → 기존 트랙과 IoU 매칭 (등속 예측 기반, 속도는 지수 블렌딩)
  - 저신뢰 탐지 → 미매칭 트랙과 2차 연결 (모션 블러·옆얼굴 프레임 복구의 핵심). 저신뢰 단독 탐지는 새 트랙을 만들지 않음
- [x] 트랙 후처리 파이프라인
  - [x] 정방향 + 역방향 패스 트랙 합집합 (`buildBidirectionalTracks`) — 저신뢰로 시작하는 구간을 역방향 패스가 복구하는 회귀 테스트 포함
  - [x] 트랙 내 탐지 공백 선형 보간 (`interpolateGaps`, 최대 공백 파라미터화)
  - [x] 좌표 시간축 스무딩 (`smoothTrack`) — 스무딩 결과와 원 탐지 박스의 합집합을 취해 "스무딩이 탐지 영역을 벗어나지 않음" 불변식 보장
  - [x] 트랙 시작/끝 여유 프레임 연장 (`extendTrackEnds`, 영상 경계 클램프)
- [x] 단위 테스트 10건 (`tests/test_tracking.cpp`, `redactly_tracking_tests` 타깃): 등속 이동, 3프레임 미탐지 보간, 한계 초과 공백 미보간, 저신뢰 연장/미생성, 두 객체 교차, 양방향 병합 무중복, 스무딩 불변식, 경계 클램프 등

**완료 기준 충족**: 합성 시퀀스 테스트 전체 통과(ctest). 미정: 칼만 필터로의 교체 여부는 실영상 검증(Phase 5) 결과로 판단 — 현재 등속+지수 블렌딩으로 충분하면 유지.

## Phase 3 — 비디오 I/O

- [ ] FFmpeg/FFprobe 발견·검증 계층
  - 번들 우선(플랫폼별 패키징 포함), 없으면 시스템 PATH 탐색
  - 실행 파일 무결성: 번들본은 체크섬 검증, 시스템본은 버전 확인 + 사용자 고지
- [ ] 디코드: `ffmpeg → rawvideo pipe → cv::Mat` 스트리밍 (전체 프레임 메모리 적재 금지)
- [ ] 인코드: `cv::Mat → rawvideo pipe → ffmpeg` (H.264/HEVC, 원본 오디오 스트림 `-c:a copy` 패스스루)
- [ ] 메타데이터 정책
  - 기본: 컨테이너 메타데이터 제거 (`-map_metadata -1`), GPS 좌표 atom 제거 확인
  - 회전 메타데이터는 픽셀에 적용 후 제거 (탐지 좌표계와 표시 좌표계 불일치 방지)
- [ ] 엣지 케이스 처리 방침 명문화
  - 가변 프레임레이트(VFR): v1은 CFR로 변환, 릴리스 노트에 명시
  - 10-bit/HDR: v1은 미지원으로 명확히 거부 (조용한 색 손상보다 나음)
  - 손상 파일·중도 실패: 부분 출력물 삭제, 파일 단위 오류 집계(사진과 동일 정책)
- [ ] 원자적 출력: 임시 파일에 완성 후 rename (기존 `atomicImwrite` 정책 준수)

**완료 기준**: 대표 샘플(H.264/HEVC, MP4/MOV, 오디오 포함, 회전 메타데이터)에서 왕복 인코딩 후 오디오 싱크·화질·재생 호환성 수동 검증 체크리스트 통과.

## Phase 4 — 파이프라인 통합 + UX

- [ ] `VideoProcessor` (2-pass 오케스트레이션)
  - 1차 패스: 스트리밍 디코드 → 프레임별 탐지(설정 간격 1–3프레임) → 트랙 구축
  - 2차 패스: 재디코드 → 트랙 기반 모자이크 적용(`Mosaic.cpp` 재사용) → 인코드
- [ ] `ProcessorWorker` 확장: 파일 타입에 따라 이미지/동영상 경로 분기, `RunOutcome` 집계 통합
- [ ] 취소: 기존 atomic `cancelled_`를 프레임 루프에서 확인, FFmpeg 서브프로세스 종료 + 부분 출력물 정리
- [ ] 진행률: 파일 단위 → 프레임 단위 2단계 표시 (패스 구분 포함, ETA)
- [ ] 미탐지 경고 정책 승계: 트랙이 0개인 동영상은 완료 요약에 경고 집계 (코드리뷰 치명 #4와 동일 원칙)
- [ ] 입력 목록·드롭 핸들러·`ImageScanner`에 동영상 확장자(.mp4 .mov .m4v) 추가
- [ ] 동영상 입력 + 리뷰 모드 조합: v1은 "동영상은 리뷰 없이 처리됨" 안내

**완료 기준**: 실사용 시나리오(이미지+동영상 혼합 배치, 중도 취소, 실패 파일 포함) 통합 테스트 통과.

## v1 범위 제외 (후속 릴리스 후보)

- 트랙 단위 타임라인 리뷰 UI (트랙 삭제/추가/구간 조정) — "이 사람은 모자이크 제외"의 자연스러운 진입점: 트랙 = 영상 내 동일 인물이므로 트랙 하나를 제외하면 얼굴 인식 모델 없이도 특정 인물 제외가 됨
- 인물 자동 제외 (얼굴 인식 임베딩) — 기준 사진으로 "항상 나는 제외" 자동화. 얼굴 인식 모델(ArcFace류) 추가 필요, 오인식 시 타인 얼굴이 노출되는 리스크가 있어 트랙 단위 다수결 판정 전제. SCRFD bnkps의 랜드마크 출력을 얼굴 정렬에 재사용 가능
- 픽셀 정밀 마스킹 (세그멘테이션/SAM2 계열) — 사각형 대신 얼굴 윤곽만 가림. 신원 구분과는 무관한 마스킹 모양 개선이며 별도 릴리스급
- 10-bit/HDR·VFR 보존, 하드웨어 인코딩(VideoToolbox/NVENC)
- Linux GPU 가속(CUDA/ROCm), Windows DirectML 동봉 패키징 전환
- 실시간 미리보기·스트림 입력
